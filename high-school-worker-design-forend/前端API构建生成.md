针对你们的大学生职业规划智能体项目，前端用户界面（UI/UX）的设计需要兼顾**信息清晰度**、**交互流畅性**和**大赛展示效果**。以下是基于你们后端功能（画像、规划、简历优化、模拟面试）的分层设计建议，采用现代前端架构（Vue/React）并适配 go-zero 的流式 API。

---

## 一、整体信息架构（IA）

建议采用 **底部导航栏（Tab Bar）** 式布局，符合大学生移动端使用习惯，同时桌面端也能适配。分为四个核心模块：

| 模块 | 英文 | 核心任务 |
|------|------|----------|
| 首页 | Home | 用户画像、职业赛道推荐、快速开始 |
| 规划 | Plan | 技能树、时间轴、差距分析 |
| 简历 | Resume | 简历解析、优化建议、双版本对比 |
| 面试 | Interview | 模拟面试（大厂/国企双模式）、历史记录 |

另外增加 **个人中心** 管理个人资料、历史记录、设置。

---

## 二、页面层次与交互流程

### 1. 首页（引导与画像）
- **顶部**：欢迎语 + 用户头像（未登录则显示引导）
- **中间**：
  - **快速诊断**：简洁的卡片式问卷（年级、专业、求职意向：大厂技术岗 / 国企研发岗），用于初步画像。
  - **职业倾向测试**：霍兰德兴趣测试入口（可选，可做成简短版）。
- **底部**：展示智能体生成的 **一句话职业建议**（基于已填信息动态刷新）。
- **交互**：用户填写基本信息后，点击“生成我的规划”跳转到规划页，同时后端创建用户画像。

### 2. 规划页（核心价值展示）
- **双赛道切换**：顶部 Tabs 或分段选择器（大厂技术岗 / 国企研发岗），切换时整个页面数据刷新。
- **技能树图谱**：使用 ECharts 或 AntV 绘制可交互的树状图，节点颜色表示掌握程度（未掌握、学习中、已掌握），点击节点可查看学习资源。
- **时间轴规划**：按学期展示关键节点（考证、实习、项目、竞赛），支持编辑和勾选完成状态。
- **差距分析卡片**：展示与目标岗位的匹配度（进度条），并列出待补足项（如“缺少 JVM 调优经验”）。
- **操作区**：按钮“刷新规划”（重新调用 AI 生成）、“导出报告”（PDF 或图片）。

### 3. 简历页（亮点功能）
- **上传区域**：支持拖拽或点击上传 PDF/Word 简历，显示解析进度。
- **双版本对比展示**：
  - 左侧原始简历内容（高亮待优化处）
  - 右侧优化建议（分点展示）
- **一键优化**：点击后展示优化后的简历（可切换“大厂版”/“国企版”），并支持复制或下载。
- **交互细节**：使用流式输出显示优化过程，增强科技感。

### 4. 面试页（模拟）
- **模式选择**：两个大按钮“大厂技术面”与“国企综合面”，点击后进入对话界面。
- **对话界面**：
  - 上方显示面试官虚拟形象或提示
  - 聊天区域展示面试问答（用户输入+AI回复）
  - 底部输入框支持文本输入
  - 每个问题回答后，AI 自动给出评分和改进建议（可折叠）
- **历史记录**：列表展示以往模拟面试，可重新回放。

### 5. 个人中心
- 用户基本信息展示与编辑
- 历史规划、简历优化记录、面试记录列表
- 设置（如切换主题、清除缓存）

## 三、前端技术栈建议

| 类别 | 推荐方案 | 理由 |
|------|----------|------|
| 框架 | Vue 3 + Vite 或 React + Next.js | 快速开发，组件生态丰富 |

| 网络请求 | Axios + SSE 原生 EventSource | 处理普通 API 和流式响应 |
| 图表 | ECharts / AntV G6 | 技能树、时间轴可视化 |
| 文件上传 | 使用 FormData + Axios | 简历解析 |
| 构建 | Vite | 开发热更新快，打包体积小 |

## 四、与后端流式 API 的对接设计

前面讨论过，后端将提供 SSE 流式接口（如 `/api/v1/jobs/generate-stream`）。前端需做好以下适配：

### 1. 使用 EventSource 接收流式内容
```javascript
// 例如在规划页生成规划时
const eventSource = new EventSource(`/api/v1/plan/generate-stream?userId=${userId}&track=bigtech`);
eventSource.onmessage = (event) => {
    const chunk = event.data;
    // 将 chunk 追加到显示区域（如规划文本）
    appendToPlan(chunk);
};
eventSource.onerror = () => eventSource.close();
```

### 2. 处理跨域与鉴权
如果使用 cookies 或 Authorization header，EventSource 无法直接添加自定义 Header。解决方案：
- 将 token 放在 URL 查询参数中（注意安全性，开发环境可接受）
- 或使用 fetch + ReadableStream 手动处理 SSE（更灵活，可添加 Header）

```javascript
const response = await fetch('/api/v1/plan/generate-stream', {
    headers: { 'Authorization': `Bearer ${token}` }
});
const reader = response.body.getReader();
const decoder = new TextDecoder();
while (true) {
    const { value, done } = await reader.read();
    if (done) break;
    const chunk = decoder.decode(value);
    // 解析 SSE 格式 data: xxx\n\n
    // 提取内容并渲染
}
```

### 3. 中断与错误处理
在用户切换页面或取消时，主动关闭连接（调用 `eventSource.close()` 或中止 fetch）。

## 五、演示与评委体验优化

1. **首屏加载效果**：在加载规划、简历优化等耗时操作时，展示骨架屏或加载动画，配合流式输出形成“实时生成”的视觉感受。
2. **数据预填充**：演示时可预置一个虚拟学生档案，减少评委操作步骤，直接展示核心功能。
3. **多端适配**：确保在笔记本大屏和手机屏幕上都能良好显示（使用响应式布局）。
4. **关键数据可视化**：将“技能匹配度”、“学习进度”等用图表直观呈现，体现技术深度。

## 六、开发建议与优先级

- **第一阶段**：完成首页 + 规划页的基础 UI 和静态展示，接入流式 API 实现文本生成。
- **第二阶段**：实现简历上传与双版本优化界面，对接简历解析接口。
- **第三阶段**：完成模拟面试对话界面，对接面试接口。
- **最后**：打磨个人中心、历史记录、导出功能，完善交互细节。

---

## 七、总结

用户界面层次设计的核心是**简洁、聚焦、高效**。通过清晰的信息架构和流畅的交互（尤其是流式输出的打字机效果），能让评委快速理解产品的价值。前端技术选型应以稳定和快速实现为主，确保在大赛前完成所有功能的联调与演示优化。
