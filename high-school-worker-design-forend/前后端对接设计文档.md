## 四、SSE 流式接口规范

### 通用约定

1. **请求头要求**
   - `Content-Type: application/json`
   - `Authorization: Bearer <token>` (必须携带)

2. **响应格式**
   ```
   data: <json string>\n\n
   data: <json string>\n\n
   data: [DONE]\n\n
   ```

3. **数据帧类型**
   - `text`: 普通文本内容 (如规划文本、面试追问)
   - `json`: JSON 对象 (如评分、反馈)
   - `[DONE]`: 流式结束标记

### 前端 SSE 封装

```typescript
// src/utils/sse.ts
export async function fetchSSE(
  url: string, 
  options: RequestInit, 
  onMessage: (data: string) => void,
  onDone?: () => void
) {
  const response = await fetch(url, options);
  const reader = response.body!.getReader();
  const decoder = new TextDecoder();
  let buffer = '';

  while (true) {
    const { value, done } = await reader.read();
    if (done) break;
    buffer += decoder.decode(value, { stream: true });
    const lines = buffer.split('\n');
    for (let i = 0; i < lines.length - 1; i++) {
      const line = lines[i].trim();
      if (line.startsWith('data:')) {
        const data = line.slice(5).trim();
        if (data === '[DONE]') {
          reader.cancel();
          onDone?.();
          return;
        }
        onMessage(data);
      }
    }
    buffer = lines[lines.length - 1];
  }
}
```

## 五、CORS 配置要求

后端需配置以下 CORS 策略：

| 配置项 | 值 |
|--------|-----|
| Allow Origins | `http://localhost:5173` (开发) |
| Allow Methods | GET, POST, PUT, DELETE, OPTIONS |
| Allow Headers | Content-Type, Authorization |
| Allow Credentials | true |

## 六、错误响应格式

```json
{
  "code": 1001,
  "msg": "错误信息描述"
}
```

**状态码约定**：
- `code: 0` - 成功
- `code: 1001` - 参数错误
- `code: 1002` - 认证失败
- `code: 1003` - 权限不足
- `code: 500` - 服务器内部错误

## 八、联调注意事项

1. **认证**：除 `/user/login` 和 `/user/register` 外，所有接口需携带 token
2. **流式中断**：页面切换时前端会中止 SSE 连接，后端需正确处理
3. **文件上传**：简历文件需转为 base64 字符串传递
4. **赛道切换**：大厂/国企通过 URL query 参数或请求体传递
