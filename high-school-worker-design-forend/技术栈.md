## 七、流式对接示例代码（核心）

由于 EventSource 无法携带自定义 Header，采用 fetch + ReadableStream 处理 SSE：

```typescript
// utils/sse.ts
export async function fetchSSE(url: string, options: RequestInit, onMessage: (data: string) => void) {
  const response = await fetch(url, options);
  const reader = response.body!.getReader();
  const decoder = new TextDecoder();
  let buffer = '';

  while (true) {
    const { value, done } = await reader.read();
    if (done) break;
    buffer += decoder.decode(value, { stream: true });
    // SSE 数据格式：data: {json}\n\n
    const lines = buffer.split('\n');
    for (let i = 0; i < lines.length - 1; i++) {
      const line = lines[i].trim();
      if (line.startsWith('data:')) {
        const data = line.slice(5).trim();
        if (data === '[DONE]') {
          reader.cancel();
          return;
        }
        onMessage(data);
      }
    }
    buffer = lines[lines.length - 1];
  }
}
```

在组件中使用：

```tsx
const handleGenerate = async () => {
  const token = localStorage.getItem('token');
  await fetchSSE('/api/v1/plan/generate-stream', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${token}`,
    },
    body: JSON.stringify({ userId, track }),
  }, (chunk) => {
    // 实时追加到显示区域
    setPlanText(prev => prev + chunk);
  });
};
```

## 九、开发流程与演示优化

1. **并行开发**：使用 Mock Service Worker (MSW) 模拟后端接口，前后端独立开发，最后联调。
2. **状态持久化**：用户画像、对话历史等使用 `localStorage` 或 IndexedDB（配合 `idb-keyval`）缓存，提升演示流畅度。
3. **性能优化**：
   - 技能树使用 `React.memo` 避免重绘。
   - 流式输出时使用 `useRef` 避免频繁 setState 导致渲染卡顿，可合并批次更新（例如每 50ms 合并一次）。
4. **移动端适配**：使用 `viewport` 配合 Tailwind 响应式类，确保在手机和 PC 演示时都美观。

## 十、总结

这套技术栈组合（React + Vite + Ant Design + Tailwind + Zustand + AntV G6 + ECharts）具备以下优势：

- **现代化**：Vite + React 18 + TypeScript 是当前主流选型，能体现团队紧跟技术趋势。
- **高效**：Ant Design 提供大量现成组件，节省开发时间；Tailwind 快速定制独特样式。
- **可视化突出**：G6 和 ECharts 足以支撑技能树、时间轴等复杂图表，给评委留下深刻印象。
- **流式体验佳**：自行封装 SSE 请求，实现“打字机”效果，提升交互流畅度。

在开发时，可以优先完成**首页 + 规划页**的静态 UI 和流式对接，快速产出可演示的 MVP，再逐步增加简历和面试功能。最终利用容器化（Docker）将前端打包为 Nginx 镜像，与后端一同部署，方便现场展示。

如果需要具体的组件实现细节（如技能树与后端数据的联动），可以继续深入探讨。
