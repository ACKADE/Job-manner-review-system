## API接口列表

### 1. POST /api/v1/interview/start
启动面试会话

### 2. POST /api/v1/interview/chat-stream
面试对话（SSE流式输出）

### 3. GET /api/v1/interview/history
获取面试历史

### 4. GET /api/v1/interview/:id
获取面试详情

### 5. GET /api/v1/interview/:id/report
获取面试报告

### 6. POST /api/v1/interview/:id/end
结束面试会话

### 7. DELETE /api/v1/interview/:id
删除面试会话

## 核心特性

### 1. 流式AI输出
- 使用SSE (Server-Sent Events) 实现实时流式输出
- 支持长连接，保持上下文连贯
- 模拟打字效果，提升用户体验

### 2. 智能评分系统
- AI自动对用户回答进行评分（0-100分）
- 实时反馈和建议
- 多维度评分（技术、沟通、逻辑、自信）

### 3. 自动报告生成
- 面试结束时自动生成评估报告
- 包含优势和劣势分析
- 提供具体改进建议

### 4. 鉴权机制
- 所有API都需要JWT Token鉴权
- 支持Header和URL参数两种Token传递方式
- 用户只能访问自己的面试记录

### 5. 数据持久化
- 完整的对话历史记录
- 面试会话状态管理
- 评估报告存储

执行数据库迁移脚本：
```bash
cd sql/migrations
./run-migration.sh
```

或手动执行：
```bash
mysql -u username -p database_name < sql/migrations/004_add_interview_tables.sql
```

### 2. 配置API密钥
在配置文件中设置DeepSeek API密钥：
```yaml
AI:
  ApiKey: "your-deepseek-api-key"
  Model: "deepseek-chat"
  BaseURL: "https://api.deepseek.com/v1"
  Timeout: 60
```

### 3. 启动服务
```bash
./start.sh
```

### 4. API测试

## 注意事项

1. **API密钥**：测试环境使用硬编码密钥，生产环境请从配置文件读取
2. **数据库迁移**：执行迁移前请备份数据库
3. **SSE超时**：流式输出超时设置为2分钟
4. **并发限制**：建议限制并发面试会话数量
