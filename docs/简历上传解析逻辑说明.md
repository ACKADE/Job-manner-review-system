# 简历上传解析逻辑说明

## 一、整体流程

```
用户上传简历 → 前端处理 → 后端接收 → 文件解析 → AI分析 → 数据保存 → 返回结果 → 前端展示
```

#### 步骤2: 文件转 Base64
```typescript
const fileToBase64 = (file: File): Promise<string> => {
  return new Promise((resolve, reject) => {
    const reader = new FileReader();
    reader.readAsDataURL(file);
    reader.onload = () => {
      const base64 = (reader.result as string).split(',')[1];
      resolve(base64);
    };
  });
};
```

#### 步骤3: 调用后端 API

### 2.2 后端处理流程

**文件位置**: `internal/logic/studentlogic.go`

#### 步骤1: 接收和验证
```go
func (l *UploadResumeLogic) UploadResume(req *types.ResumeUploadReq) (*types.StudentResp, error) {
  // 1. 验证参数
  if req.FileContent == "" || req.FileName == "" {
    return &types.StudentResp{Code: errors.CodeInvalidParams, Msg: "fileContent and fileName are required"}
  }

  // 2. 验证文件格式
  if !strings.HasSuffix(fileName, ".pdf") && !strings.HasSuffix(fileName, ".docx") {
    return &types.StudentResp{Code: errors.CodeInvalidParams, Msg: "unsupported file format"}
  }

  // 3. 验证文件大小
  if int64(len(req.FileContent)) > maxBase64Size {
    return &types.StudentResp{Code: errors.CodeInvalidParams, Msg: "file size exceeds 10MB limit"}
  }
}
```

#### 步骤2: 解码 Base64
```go
fileData, err := base64.StdEncoding.DecodeString(req.FileContent)
if err != nil {
  return &types.StudentResp{Code: errors.CodeInvalidParams, Msg: "invalid base64 content"}
}
```

#### 步骤3: 保存临时文件
```go
tempFile := filepath.Join(tempDir, fmt.Sprintf("resume_%d%s", time.Now().UnixNano(), filepath.Ext(req.FileName)))
```

#### 步骤4: 提取文本内容


#### 步骤5: 调用 AI API 解析
```go

  existingStudent.Education = sql.NullString{String: profile.Education, Valid: true}
  existingStudent.Major = sql.NullString{String: profile.Major, Valid: true}
  existingStudent.Skills = sql.NullString{String: string(skillsJSON), Valid: true}
  existingStudent.Suggestions = sql.NullString{String: string(suggestionsJSON), Valid: len(profile.Suggestions) > 0}
  existingStudent.ResumeContent = sql.NullString{String: resumeText, Valid: true}
  existingStudent.CompletenessScore = profile.Completeness
  existingStudent.CompetitivenessScore = profile.Competitiveness
  existingStudent.UpdatedAt = time.Now().Unix()
```

## 五、性能分析

### 5.1 各阶段耗时

| 阶段 | 耗时 | 说明 |
|------|------|------|
| 文件上传 | 2-5s | 取决于文件大小和网络速度 |
| Base64 解码 | <1s | 本地操作 |
| 文本提取 | 5-10s | PDF/DOCX 解析 |
| AI 分析 | 40-60s | AI API 响应时间 |
| 数据库保存 | 1-2s | 插入/更新操作 |
| **总计** | **50-80s** | |

### 5.2 优化建议

1. **文本提取优化**: 使用更快的 PDF/DOCX 解析库
2. **AI 响应优化**: 优化 Prompt 减少 Token 消耗
3. **并发处理**: 文本提取和 AI 调用可以并行
4. **缓存机制**: 相同简历内容可以缓存解析结果
5. **流式响应**: 实时返回解析进度

## 六、错误处理

### 6.1 前端错误

```typescript
try {
  const response = await studentApi.uploadResume({
    fileContent: base64Content,
    fileName: file.name,
  });
  
  if (response.code === 0) {
    // 成功
    setProfile(response.data);
    setParsed(true);
  } else {
    // 业务错误
    setError(response.msg || '解析失败');
  }
} catch (err) {
  // 网络错误
  let errorMsg = '上传失败，请检查网络连接';
  
  if (err.response?.data) {
    errorMsg = err.response.data.msg || errorMsg;
  } else if (err.message) {
    errorMsg = err.message;
  }
  
  // 特殊处理 401 错误
  if (err.response?.status === 401) {
    errorMsg = '登录已过期，请重新登录';
  }
  
  setError(errorMsg);
}
```

## 七、常见问题

### 7.1 为什么显示"未提取"？

**原因**: 前端使用了错误的字段来判断完整度

**修复**: 
```typescript
// 之前（错误）
<span>{profile.resumeContent ? '已提取' : '未提取'}</span>

// 现在（正确）
<span>{profile.completeness}%</span>
```

### 7.2 为什么建议列表为空？

**原因**: 后端没有反序列化 suggestions 字段

**修复**:
```go
// 添加 suggestions 反序列化
var suggestions []string
if student.Suggestions.Valid {
  json.Unmarshal([]byte(student.Suggestions.String), &suggestions)
}

// 返回时包含 suggestions
Data: &types.StudentProfile{
  // ... 其他字段
  Suggestions: suggestions,
}
```

### 7.3 为什么超时？

**原因**: 前端 timeout 只有 30 秒，但完整流程需要 50-80 秒

**修复**: 将 timeout 增加到 120 秒
