# 面试模块实现文档

## 概述

本文档描述了面试模块的流式输出实现方案，采用传统的打字问答交互方式，实时显示AI回复并自动评分。

## 核心功能

### 1. 流式AI输出
- 使用Server-Sent Events (SSE)格式实现流式响应
- 实时显示AI回复内容，模拟打字效果
- 支持长时间对话，保持上下文连贯性

### 2. 打字问答交互
- 命令行界面，用户输入回答
- AI实时流式输出回复
- 支持连续多轮对话

### 3. 智能评分系统
- AI自动对用户回答进行评分（0-100分）
- 提供具体的反馈建议
- 根据回答质量调整后续问题难度

### 4. 对话历史管理
- 在内存中保存完整对话历史
- 支持上下文引用
- 记录面试时长和评分统计

## 核心算法

### 流式输出处理

```go
func (ai *InterviewAI) InterviewStream(
    ctx context.Context,
    session *InterviewSession,
    userMessage string,
) (<-chan string, <-chan error, <-chan InterviewQuestion) {
    // 1. 创建通信通道
    contentChan := make(chan string, 100)
    errChan := make(chan error, 1)
    questionChan := make(chan InterviewQuestion, 1)

    go func() {
        defer close(contentChan)
        defer close(errChan)
        defer close(questionChan)

        // 2. 构建消息历史
        messages := append([]ChatMessage{systemMsg}, session.History...)

        // 3. 发送流式请求

        // 4. 读取流式响应
        reader := bufio.NewReader(resp.Body)
        for {
            line, _ := reader.ReadString('\n')
            if strings.HasPrefix(line, "data: ") {
                data := strings.TrimPrefix(line, "data: ")
                // 解析并发送到channel
                contentChan <- content
            }
        }

        // 5. 解析完整JSON响应
        var response struct {
            Question string  `json:"question"`
            Score    float64 `json:"score"`
            Feedback string  `json:"feedback"`
        }
        json.Unmarshal(fullResponse, &response)

        // 6. 更新会话状态
        session.Score = response.Score
        session.Feedback = response.Feedback

        // 7. 发送下一个问题
        questionChan <- InterviewQuestion{...}
    }()

    return contentChan, errChan, questionChan
}
```

## 性能优化

### 1. 流式输出优化

- **缓冲区大小**: 使用缓冲channel (`chan string, 100`) 减少阻塞
- **定时刷新**: 每50ms刷新一次输出缓冲区
- **并发处理**: 使用goroutine异步处理API响应

### 2. 内存管理

- **历史记录限制**: 限制保存的对话历史长度
- **及时关闭**: 使用defer确保channel和连接正确关闭
- **错误处理**: 完善的错误处理机制，避免资源泄漏

### 3. 网络优化

- **超时设置**: 60秒超时，避免长时间等待
- **连接复用**: 使用HTTP客户端连接池
- **流式传输**: 减少网络传输延迟

## 错误处理

### 常见错误及处理

| 错误类型 | 错误信息 | 处理方式 |
|---------|---------|---------|
| API密钥未设置 | DEEPSEEK_API_KEY not set | 提示用户设置环境变量 |
| API密钥无效 | Invalid API key | 验证密钥格式，提示重新设置 |
| 网络连接失败 | dial tcp: timeout | 检查网络连接，重试机制 |
| API响应错误 | status=429 Too Many Requests | 限流处理，延迟重试 |
| JSON解析失败 | parse response failed | 记录错误日志，提示用户重新输入 |
| 流式读取失败 | read stream failed | 重试或优雅降级 |

### 错误处理代码示例

```go
// 1. API密钥验证
if apiKey == "" {
    return fmt.Errorf("DEEPSEEK_API_KEY not set")
}

// 2. HTTP请求错误处理
resp, err := client.Do(req)
if err != nil {
    return fmt.Errorf("http request failed: %v", err)
}
defer resp.Body.Close()

// 3. API状态码检查
if resp.StatusCode != http.StatusOK {
    body, _ := io.ReadAll(resp.Body)
    return fmt.Errorf("API error: status=%d, body=%s", resp.StatusCode, string(body))
}

// 4. JSON解析错误处理
if err := json.Unmarshal(data, &response); err != nil {
    return fmt.Errorf("parse response failed: %v", err)
}

// 5. 流式读取错误处理
for {
    line, err := reader.ReadString('\n')
    if err != nil {
        if err != io.EOF {
            errChan <- fmt.Errorf("read stream failed: %v", err)
        }
        break
    }
}
```
