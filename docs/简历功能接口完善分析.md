## 二、缺失功能分析

### 2.1 优化建议功能

#### 需求描述
用户上传简历后，系统应该提供具体的简历优化建议，帮助用户改进简历质量。

#### 当前状态
- ❌ 后端未实现优化建议生成
- ❌ 前端未实现优化建议展示
- ❌ 数据库未存储优化建议

#### 实现方案

**方案1：扩展StudentProfile结构（推荐）**

在`StudentProfile`中添加`suggestions`字段：

```go
type StudentProfile struct {
    // ... 现有字段
    Suggestions []string `json:"suggestions"`
}
```

**方案2：创建独立的优化建议表**

```sql
CREATE TABLE IF NOT EXISTS resume_suggestions (
    id BIGINT NOT NULL AUTO_INCREMENT,
    student_id BIGINT NOT NULL,
    suggestions JSON,
    created_at BIGINT NOT NULL,
    PRIMARY KEY (id),
    KEY idx_student (student_id)
)
```

**AI Prompt优化**

需要在`GenerateStudentProfile`的Prompt中添加优化建议生成的指令：

```
{
  "suggestions": [
    "建议1：具体描述",
    "建议2：具体描述",
    "建议3：具体描述"
  ]
}
```

### 2.2 历史简历解析记录功能

#### 需求描述
用户可以查看历史简历解析记录，包括：
- 解析时间
- 解析结果
- 优化建议
- 完整度变化

#### 当前状态
- ❌ 后端未实现历史记录查询接口
- ❌ 前端未实现历史记录展示
- ❌ 数据库未存储历史记录

#### 实现方案

**创建历史记录表**

```sql
CREATE TABLE IF NOT EXISTS resume_parse_history (
    id BIGINT NOT NULL AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    student_id BIGINT,
    resume_content TEXT,
    parsed_profile JSON,
    suggestions JSON,
    completeness_score FLOAT,
    competitiveness_score FLOAT,
    created_at BIGINT NOT NULL,
    PRIMARY KEY (id),
    KEY idx_user (user_id),
    KEY idx_student (student_id),
    KEY idx_created (created_at)
)
```

**新增API接口**

```go
// 获取用户简历解析历史
GET /api/v1/students/resume/history

// 获取特定解析记录详情
GET /api/v1/students/resume/history/:id

// 删除历史记录
DELETE /api/v1/students/resume/history/:id
```

### 3.2 需要新增的接口

#### 1. 获取简历优化建议

```go
// API定义
type GetSuggestionsReq {
    StudentId int64 `json:"studentId,validate:"required,gt=0"`
}

type SuggestionsResp {
    Code int `json:"code"`
    Msg  string `json:"msg"`
    Data struct {
        Suggestions []string `json:"suggestions"`
        Priority    string   `json:"priority"`  // high/medium/low
        Category    string   `json:"category"`  // content/format/experience
    } `json:"data,optional"`
}

// 路由
GET /api/v1/students/:id/suggestions
```

#### 2. 获取简历解析历史

```go
// API定义
type ResumeHistoryReq {
    Page     int `form:"page,default=1"`
    PageSize int `form:"pageSize,default=10"`
}

type ResumeHistoryRecord struct {
    Id                  int64           `json:"id"`
    ResumeFileName      string          `json:"resumeFileName"`
    ParsedProfile       *StudentProfile `json:"parsedProfile"`
    Suggestions         []string        `json:"suggestions"`
    CompletenessScore   float64         `json:"completenessScore"`
    CompetitivenessScore float64        `json:"competitivenessScore"`
    CreatedAt           int64           `json:"createdAt"`
}

type ResumeHistoryResp {
    Code int                   `json:"code"`
    Msg  string                `json:"msg"`
    Data struct {
        Total int64                `json:"total"`
        List  []ResumeHistoryRecord `json:"list"`
    } `json:"data,optional"`
}

// 路由
GET /api/v1/students/resume/history
```

#### 3. 查看历史解析详情

```go
// API定义
type HistoryDetailResp {
    Code int `json:"code"`

        Record   *ResumeHistoryRecord `json:"record"`

        Parsed   *StudentProfile      `json:"parsed"`   // 解析后的档案
    } `json:"data,optional"`
}

// 路由
GET /api/v1/students/resume/history/:id
```

#### 4. 删除历史记录

```go
// 路由
```

1. **AI Prompt优化**
   - 确保优化建议具体、可操作
   - 建议数量控制在3-5条
   - 建议分类（内容、格式、经验等）

2. **数据库性能**
   - 为历史记录表添加合适的索引
   - 考虑历史记录的存储策略（保留多长时间）
   - 定期清理旧的历史记录

3. **用户体验**
   - 解析时间较长时提供进度提示
   - 优化建议要清晰易懂
   - 历史记录要有时间戳和版本标识

4. **安全性**
   - 限制历史记录的查询频率
   - 验证用户权限
   - 加密敏感信息

## 八、总结

当前简历上传和解析功能已经基本完成，可以解析简历并生成学生档案。但是还缺少以下核心功能：

1. **优化建议**：需要在AI解析时生成并返回具体的简历优化建议
2. **历史记录**：需要记录每次解析的历史，方便用户查看和对比
3. **数据持久化**：解析结果需要保存到数据库

建议按照上述实现步骤，优先完成P0级别的功能，确保核心功能可用，然后再逐步完善P1和P2级别的功能。
