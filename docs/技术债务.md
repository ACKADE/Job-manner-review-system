## 🔴 高优先级

### 1. Logic层实现分散✅

**问题描述**:
- `internal/logic/userlogic.go` 中包含了完整的用户相关Logic实现
- `internal/logic/user/` 目录下也有对应的Logic文件（registerlogic.go、loginlogic.go等）
- 这导致代码重复和维护困难

**影响范围**:
- 代码维护困难，容易造成不一致
- 测试覆盖不完整
- 代码审查复杂

**建议解决方案**:
1. 删除 `internal/logic/userlogic.go` 中的用户相关Logic实现（RegisterLogic、LoginLogic、GetUserInfoLogic、UpdateUserInfoLogic）
2. 只保留 `internal/logic/user/` 目录下的实现
3. 统一代码组织结构，参考go-zero的标准目录结构
4. 更新所有导入路径

**预估工作量**: 2-3小时

---

### 2. 参数验证未完全生效✅

**问题描述**:
- 已在API定义中添加了验证规则（validate标签）
- 已创建验证中间件 `internal/middleware/validation.go`
- 已编写验证测试用例（22个测试全部通过）
- 但go-zero框架的`httpx.Parse`不会自动应用validator

**影响范围**:

- 参数验证功能未在生产环境中生效
- 可能接收到无效数据
- 验证测试已通过但未应用到实际请求

**建议解决方案**:
1. 在Handler中手动调用验证中间件
2. 或在Logic层添加验证逻辑，参考RegisterLogic中的参数检查
3. 或使用go-zero的内置验证机制（需要进一步研究）
4. 优先在用户注册、登录等关键接口中添加验证

**预估工作量**: 4-6小时

---

### 3. 数据库表结构优化✅

**问题描述**:
- `created_at` 和 `updated_at` 字段没有数据库级别的默认值
- 需要在应用层手动设置时间戳
- 已通过`InsertWithTimestamp`方法临时解决
- 每个Model都需要实现InsertWithTimestamp方法

**影响范围**:

- 依赖应用层保证数据完整性
- 如果直接操作数据库可能导致数据不一致
- 代码重复（每个Model都需要实现相同逻辑）

**建议解决方案**:
1. 修改数据库表结构，添加默认值和自动更新
2. 示例SQL：
   ```sql
   ALTER TABLE users 
   MODIFY COLUMN created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
   MODIFY COLUMN updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP;
   
   ALTER TABLE students 
   MODIFY COLUMN created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
   MODIFY COLUMN updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP;
   
   -- 对其他表也执行类似的修改
   ```
3. 简化Model层代码，移除InsertWithTimestamp方法
4. 更新数据库迁移脚本

**预估工作量**: 3-4小时

### 5. 配置管理需要改进

**问题描述**:
- 配置文件 `etc/career-api.yaml` 中包含敏感信息（AI API Key）
- JWT secret使用默认值 "your-secret-key-change-in-production"
- 配置文件已提交到git仓库

**影响范围**:
- 生产环境安全风险
- API Key可能泄露
- 不符合安全最佳实践

**建议解决方案**:
1. 使用环境变量管理敏感配置
2. 在`.gitignore`中添加配置文件
3. 提供配置示例文件 `etc/career-api.yaml.example`
4. 更新配置加载逻辑，支持环境变量覆盖
5. 示例：
   ```yaml
   # etc/career-api.yaml.example
   Auth:
     AccessSecret: ${JWT_SECRET:your-secret-key-change-in-production}
   
   AI:
     ApiKey: ${AI_API_KEY:your-api-key-here}
   ```
6. 更新文档说明配置方式

**预估工作量**: 2-3小时
