# 前后端对接计划

## 概述

本文档描述高中职业规划助手系统前端与后端的对接计划。

## 当前状态分析

### 前端现状
- 构建工具：Vite + React + TypeScript
- 位置：`high-school-worker-design-forend/`
- 数据状态：所有数据均为 mock
- 已配置：
  - 完整的 API client（axios）
  - TypeScript 类型定义
  - JWT 认证拦截器
- 主要页面：
  - Home：首页导航
  - Profile：用户档案
  - Plan：职业规划
  - Resume：简历管理
  - Interview：面试模拟

### 后端现状
- 技术栈：Go + go-zero
- 已实现模块：
  - User：用户认证
  - Student：学生档案
  - Job：职位管理
  - Match：匹配分析
  - Report：报告生成
  - Graph：职业图谱
  - Interview：面试模拟（handler已实现但未注册路由）
- 已注册路由：除 Interview 外的所有接口

**SSE 流式响应处理：**
```typescript
// 前端需要实现 SSE 客户端
function connectSSE(url: string, onMessage: (data: any) => void) {
  const eventSource = new EventSource(url);
  eventSource.onmessage = (event) => {
    const data = JSON.parse(event.data);
    onMessage(data);
  };
  return eventSource;
}
```

## 需要注意的问题

### 1. 类型定义差异
前后端部分类型定义不一致，需要统一：

- **skills**：前端是 `string[]`，后端是 `StudentSkill[]`
- **certificates**：前端是 `string[]`，后端是 `StudentCert[]`
- **gapAnalysis**：结构不完全一致

### 2. SSE 流式响应
- Report 和 Interview 模块使用 Server-Sent Events
- 前端需要实现 SSE 客户端，已在 `src/utils/sse.ts` 和 `src/hooks/useSSE.ts` 中有基础实现
- 需要处理断线重连和错误恢复

### 3. 文件上传
- 简历上传需要处理：
  - 文件转 base64（前端已实现）
  - 大文件上传进度显示
  - 文件类型和大小验证

### 4. 认证中间件
- 确保需要认证的接口正确应用 JWT 中间件
- Token 过期处理（401 自动跳转已实现）

### 5. 开发环境配置
```bash
# 前端环境变量
VITE_API_BASE_URL=http://localhost:8080/api/v1

# 启动命令
cd high-school-worker-design-forend
npm run dev
```
