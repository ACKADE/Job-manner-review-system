## 1. 产品概述

### 1.1 产品定位

**职业规划智能系统**是一款职业规划智能平台，基于AI技术为学生提供个性化职业发展指导。系统解决了"不知道未来该做什么"的核心痛点，帮助学生科学认识自我、发掘职业潜能、规划职业发展路径。

### 1.2 核心价值

- **科学认知**：通过科学的评估工具帮助学生了解自己的职业兴趣和能力倾向
- **智能匹配**：基于AI算法实现人岗精准匹配，提供个性化职业推荐
- **规划指导**：生成详细的职业发展报告，提供具体的学习和提升建议
- **便捷高效**：随时随地可用的在线平台，5-10分钟即可完成核心测试

### 1.3 目标用户

- **主要用户**：高中在校学生（约2000万）
- **次要用户**：学生家长、高中教师、职业规划师
- **适用场景**：高考志愿填报、职业探索、能力提升规划

### 1.4 产品愿景

成为职业规划的**首选智能助手**，帮助千万学子科学规划人生，实现职业梦想！

## 2. 核心功能

### 2.1 职业倾向测试 🎯

**霍兰德职业倾向测试**是基于RIASEC模型的科学评估工具，帮助学生快速了解自己的职业兴趣类型。

**功能特点**：
- 18道精心设计的题目，5-10分钟即可完成
- 基于心理学理论，科学可靠
- 即时获得测试结果和职业建议
- 雷达图可视化展示兴趣分布

**测试结果包含**：
- 三位职业代码（如RIA）
- 六大类型得分明细
- 适合的职业推荐
- 详细的职业倾向描述

**适用场景**：
- 职业探索
- 高考志愿填报参考
- 专业选择辅助决策

### 2.2 简历智能解析 

上传简历即可自动提取关键信息，智能生成学生数字档案。

**功能特点**：
- 支持多种简历格式（PDF、Word等）
- AI智能识别技能、证书、实习经历
- 自动评估学生竞争力
- 生成完整的学生画像

**解析内容**：
- 基本信息（姓名、学历、专业等）
- 技能清单及熟练度评估
- 证书和认证信息
- 实习和工作经验
- 软技能评分（创新、学习、抗压、沟通、协作）

**适用场景**：
- 求职前的自我评估
- 简历优化建议
- 能力差距分析

### 2.3 智能人岗匹配 

多维度评估学生与职位的匹配度，提供精准的职业推荐。

**功能特点**：
- 多维度评分系统（技能、证书、软技能、经验）
- 整体匹配度计算（0-100分）
- 详细的差距分析
- 批量职位匹配和推荐

**匹配维度**：
- **技能匹配**：硬技能的符合度
- **证书匹配**：专业认证的相关性
- **软技能匹配**：软技能的适配度
- **经验匹配**：实习和工作经验的相关性

**输出结果**：
- 整体匹配度评分
- 各维度得分详情
- 能力差距分析
- 具体提升建议

### 2.4 职业图谱系统 

可视化的职业发展路径展示，帮助学生了解职业晋升和转行可能性。

**功能特点**：
- 晋升路径：清晰的职业发展阶梯
- 转行路径：技能迁移的可能性分析
- 相关职位：基于技能的岗位推荐
- 可视化展示，直观易懂

**图谱类型**：
- **垂直路径**：从初级到高级的晋升路线
- **横向路径**：相关职业的转行路线
- **完整图谱**：全方位的职业网络展示

**适用场景**：
- 了解职业发展前景
- 规划职业晋升路径
- 探索转行可能性

### 2.5 职业规划报告 

基于学生信息和目标职位，AI生成个性化职业发展报告。

**功能特点**：
- AI智能生成，内容个性化
- 流式输出，实时预览
- 支持历史记录查看
- 可手动编辑和优化

**报告内容**：
- **现状分析**：学生当前能力评估
- **目标定位**：目标职位画像和要求
- **差距分析**：能力差距详细分析
- **行动计划**：具体的学习和提升建议
- **资源推荐**：相关的学习资源和课程

**适用场景**：
- 制定职业发展计划
- 明确学习和提升方向
- 准备求职面试

### 2.6 面试模拟训练 

（开发中）智能对话引擎，模拟真实面试场景，帮助学生提升面试技巧。

**功能特点**：
- 多种面试模式（大厂/国企/外企）
- AI实时追问，多轮对话
- 面试表现评分和反馈
- 历史面试记录查看

**面试类型**：
- 技术面试
- 综合素质面试
- 压力面试
- 群体面试

**适用场景**：
- 求职前面试准备
- 提升面试技巧
- 模拟真实面试环境

## 5. 技术架构

### 5.1 整体架构

系统采用前后端分离的微服务架构，支持高并发和横向扩展。

```
┌─────────────────────────────────────────┐
│           用户终端（Web/Mobile）          │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│         Nginx（反向代理 + 静态资源）       │
└─────────────────────────────────────────┘
                    ↓
        ┌───────────┴───────────┐
        ↓                       ↓
┌───────────────┐       ┌───────────────┐
│   前端应用     │      │   后端API服务 │
│  (React+TS)   │       │  (Go+go-zero) │
└───────────────┘       └───────────────┘
                                ↓
                    ┌───────────┼───────────┐
                    ↓           ↓           ↓
              ┌─────────┐  ┌─────────┐  ┌─────────┐
              │  MySQL  │  │  Redis  │  │ DeepSeek│
              │  数据库  │ │  缓存   │  │  AI服务 │
              └─────────┘  └─────────┘  └─────────┘
```

### 5.2 后端架构

**分层设计**：
- **Handler层**：HTTP请求处理
- **Logic层**：业务逻辑实现
- **Model层**：数据模型定义
- **Middleware层**：中间件（认证、限流、日志等）

**微服务模块**：
- 用户认证服务
- 学生管理服务
- 职位管理服务
- 智能匹配服务
- 报告生成服务

```
    ↓

    ↓
返回Token给前端
    ↓
前端存储Token
    ↓
后续请求携带Token
```

**职业匹配流程**：
```
用户选择职位
    ↓

    ↓

    ↓

    ↓

    ↓
返回结果给用户
```

```
用户提交生成请求
    ↓

    ↓
AI生成报告内容
    ↓
流式输出给前端
    ↓
前端实时展示
    ↓
保存报告到数据库
```
