## 端口规划

为了避免与现有服务（8080、3000）冲突，建议使用以下端口分配：

| 服务 | 端口 | 说明 |
|------|------|------|
| Nginx | 80/443 | Web 服务器入口 |
| 后端 API | 8088 | API 服务（保持不变） |
| Prometheus | 9091 | 监控服务（保持不变） |
| 前端开发 | 5173 | 仅开发环境使用 |

## Nginx 配置

### 创建 Nginx 配置文件

创建 `/etc/nginx/sites-available/career-app`：

```nginx
server {
    listen 80;
    server_name your-domain.com;  # 替换为你的域名或 IP

    # 日志配置
    access_log /var/log/nginx/career-app-access.log;
    error_log /var/log/nginx/career-app-error.log;

    # 前端静态文件
    location / {
        root /var/www/career-app/dist;
        try_files $uri $uri/ /index.html;
        index index.html;
        expires 1d;
        add_header Cache-Control "public, immutable";
    }

    # 后端 API 反向代理
    location /api/ {
        proxy_pass http://localhost:8088/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 120s;
    }

    # Prometheus 监控（可选，建议仅内网访问）
    location /metrics {
        proxy_pass http://localhost:9091/metrics;
        allow 127.0.0.1;
        deny all;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        root /var/www/career-app/dist;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
```

### 启用配置

```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/career-app /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重载 Nginx
sudo systemctl reload nginx
```

## 进程管理

### 创建 Systemd 服务文件

创建 `/etc/systemd/system/career-api.service`：

```ini
[Unit]
Description=Career API Service
After=network.target mysql.service redis.service

[Service]
Type=simple
User=www-data
Group=www-data
WorkingDirectory=/opt/career-app
Environment="MYSQL_DATASOURCE=root:your_password@tcp(localhost:3306)/career_db?charset=utf8mb4&parseTime=true&loc=Local"
Environment="REDIS_HOST=localhost:6379"
Environment="REDIS_PASS=your_redis_password"
Environment="ACCESS_SECRET=your-production-secret-key"
Environment="AI_API_KEY=your-ai-api-key"
ExecStart=/opt/career-app/career-api -f /opt/career-app/etc/career-api-prod.yaml
Restart=always
RestartSec=10
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

### 启动和管理服务

```bash
# 重载 systemd 配置
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start career-api

# 设置开机自启
sudo systemctl enable career-api

# 查看服务状态
sudo systemctl status career-api

# 查看服务日志
sudo journalctl -u career-api -f

# 重启服务
sudo systemctl restart career-api
```

## 安全注意事项

### 1. 敏感信息管理

- ✅ 使用环境变量存储敏感信息
- ✅ 修改默认的 `AccessSecret`
- ✅ 修改数据库和 Redis 密码
- ✅ AI API Key 应该保密，不要提交到版本控制

### 2. HTTPS 配置

使用 Let's Encrypt 免费证书：

```bash
# 安装 Certbot
sudo apt install -y certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo certbot renew --dry-run
```

更新 Nginx 配置后，Certbot 会自动添加 SSL 配置。

### 3. 防火墙配置

```bash
# 允许 HTTP 和 HTTPS
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# 仅允许本地访问后端端口
sudo ufw deny 8088/tcp
sudo ufw deny 9091/tcp

# 启用防火墙
sudo ufw enable
```

### 4. CORS 配置

在生产环境配置中，只允许受信任的域名：

```yaml
CORS:
  Origins:
    - https://your-domain.com
    - http://your-domain.com
  # 不要使用通配符 *
```

### 5. 日志和监控

- 定期检查应用日志和 Nginx 日志
- 监控服务器资源使用情况
- 设置日志轮转，避免磁盘空间不足

### 6. 数据库安全

- 创建专用数据库用户，避免使用 root
- 配置 MySQL 只监听本地地址
- 定期备份数据库
